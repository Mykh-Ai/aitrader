# Research Ops Example

This file is a public-safe placeholder for local research operations notes.

Keep the real `research/OPS.md` local-only. It may contain:
- server access details
- host-specific runtime paths
- private runbooks and closeout procedures
- operator workflows that should not be published

Public repo guidance should reference `local-only ops materials` or a `private runbook area` instead of the private file contents.
