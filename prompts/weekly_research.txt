## Крок 1 — Знайти unprocessed runs

На сервері (`/opt/aitrader/analyzer_runs/`):
- runs БЕЗ `_processed.json` = нові, треба обробити
- runs З `_processed.json` = вже оброблені, пропустити

Також звір з `research/run_log.csv` — рядки UNPROCESSED підтверджують список.

## Крок 2 — Для кожного нового run: probe → decide → replay → record

### Probe
Перевір наявність:
- `analyzer_setups.csv`, `analyzer_setup_shortlist.csv`, `analyzer_research_summary.csv`
Порахуй: setups / shortlist rows / FormalizationEligible rows

### Decide
- FormalizationEligible == 0 → статус NO_REPLAYABLE_RULESETS, replay не запускати
- FormalizationEligible > 0 → запускати backtester (параметри в OPS.md секція 4)

### Replay (якщо eligible)
Параметри і команди — точно за `research/OPS.md`, секція 4.
Output dir: `/opt/aitrader/backtest_runs/<run_id>_routine_<YYYYMMDD>`

### Читати результат
Читати `backtest_promotion_decisions.csv`.
Для fan-out runs — кожен `derived_run_*` окремо.
Визначити один outcome state (список в OPS.md секція 5).

### Record — БЛОКУЮЧИЙ КРОК
Виконати ДО переходу до наступного run. Перевірити обидва пункти:

[ ] написаний `_processed.json` в `analyzer_runs/<run_id>/` (формат в OPS.md секція 2)
[ ] рядок доданий в `research/run_log.csv`

Якщо хоч один пункт не виконаний — не переходити до наступного run.

## Крок 3 — Перевірити ключовий project signal

Окремо відповісти:
- Чи є хоч один run з promotion_outcome = REVIEW?
- Чи є хоч один run з promotion_outcome = PROMOTE?
- Якщо ні — чесно зафіксувати: all-REJECT, research mode continues

## Крок 4 — Оновити slice analysis (якщо є нові runs)

Запустити:
    python research/slice_analysis_reclaim_context.py \
      --runs-dir /opt/aitrader/analyzer_runs

Методологію не змінювати.
Порівняти з попереднім snapshot:
- dir: `research/results/`
- naming: `reclaim_context_asymmetry_summary_<YYYY-MM-DD>.csv`
- baseline (без дати): `reclaim_context_asymmetry_summary.csv`
- знайти файл з найсвіжішою датою або baseline, якщо dated файлів ще немає

Якщо результати суттєво змінились (n, pos_rate, mean_ret дельта > noise) —
зафіксувати в новому файлі:
`research/results/reclaim_context_asymmetry_summary_<YYYY-MM-DD>.csv`
